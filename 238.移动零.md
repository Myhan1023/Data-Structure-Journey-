### 题目来源
[LeetCode 283 - Move Zeroes] (https://leetcode.cn/problems/move-zeroes/)

### 核心思想
- 如果使用暴力法（遇到0就把后面全体前移，时间复杂度高达O(n²)，面对庞大的数组十会导致超时（TLE）
- 通过==T27==的启发，这道题可以用**快慢指针**来实现时间复杂度降到O(n)

### 解题思路
- 第一阶段：存放不是0的元素
     - **快指针**：寻找所有**不等于 0** 的元素
     - **慢指针 (`slow`)**：把快指针找到的非零元素按顺序存好
     - 跑完这一遍后，数组前半部分已经全是按顺序排好的非零元素，而此时的 `slow` 指针刚好停在“有效非零数组”的下一个坑位。
- 第二阶段：一键补0
     - 从`slow`指针当前的位置开始，一直到数组的最后，全都为被清理掉的元素坑位
     - 直接一个新循环，从`slow`开始遍历，到数组末尾的所有位置均写`0`

### 代码
```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int slow=0;
        int L=nums.size();
        for(int fast=0;fast<L;fast++){
            if(nums[fast]!=0){
                nums[slow]=nums[fast];
                slow++;
            }
        }
        for(int i=slow;i<L;i++){
            nums[i]=0;
        }
    }
};
```
### 时间/空间复杂度
- **时间复杂度**：O(n) 
    - 快指针遍历一次数组，后续填零操作最多也就遍历一次剩余位置，总计跑完一趟，速度极快
- **空间复杂度**：O(1)
     - 原地修改，未开辟新的空间
