### 题目来源
[LeetCode 27 -Remove Element] (https://leetcode.cn/problems/remove-element/)

### 核心思想
- **暴力法**：利用顺序存储的空间连续特性，发现目标元素后，将其删除后，后面所有元素要向前移动一个位置
- **关键细节**：
     - `i--`：防止漏查新移动过来的元素
     - `L--`：缩减有效边界，防止循环跑到数组最原始的尽头

### 解题思路
- **1.0（错误）**：只考虑了`nums[i]=nums[i+1]`，该删除元素后面一个元素的移动，未使后面所有的元素往前移动，所以要加内层循环，让后面所有元素向前走一步
- **2.0（改进）**：加入内层循环后，`j`的范围要注意为`j<L-1`，防止数组越界
    - 如果把条件写成 `j < L`，那么 `j` 最大可以取到 `L - 1`。此时等号右边就变成了 `nums[(L-1) + 1]`，也就是去读取 **`nums[L]`**——**这直接就越界报错了**。
    - 所以，要把条件卡死在 `j < L - 1`。这样 `j` 最大只能取到 `L - 2`，等号右边最多只会读到 `nums[L-1]`，刚好安全地把最后一个元素搬过来。
- **3.0（进阶）**：==双指针法==

### 代码实现（C++）
```cpp
class Solution {
public:
    int removeElement(vector<int>& nums, int val) {
        int L=nums.size();
        for(int i=0;i<L;i++){
            if(nums[i]==val){
                for(int j=i;j<L-1;j++){
                    nums[j]=nums[j+1];
                }
                L--;
                i--;
            }
        }
        return L;
    }

};
```

### 时间/空间复杂度
- **时间复杂度**：O(n²)  双层嵌套循环         
- **空间复杂度**：O(1)  原地修改，未开辟新的空间
